# 데이터 검수 가이드

이 문서는 “니케의 모든 것” 로스터를 수동 검수할 때 사용하는 기준입니다. 외부 사이트 문장이나 공략 원문을 그대로 저장하지 않고, 확인한 사실과 직접 작성한 요약만 남깁니다.

## sourceStatus 기준

- `verified`: 공식/인게임 기본 정보가 확인됐고, 추천 메모와 점수도 관리자 검수가 끝난 상태
- `communityChecked`: 공식 기본 정보가 확인됐고, 여러 커뮤니티/도구에서 역할 평가가 대체로 일관된 상태
- `needsReview`: 기본 사용은 가능하지만 스킬작, 오버로드, 큐브, 점수, 최신 메타 검수가 필요한 상태
- `placeholder`: 이름이나 외부 목록 대조만 먼저 끝났고, 대부분의 전투 데이터가 미확인인 상태

## verified로 올리는 기준

- 한국어명, 영어명, 등급, 버스트, 클래스, 무기, 속성, 제조사가 확인됨
- 역할 태그와 `goodFor`가 실제 운용과 크게 어긋나지 않음
- `contentScores`가 모든 콘텐츠에 0~100 기준으로 입력됨
- 스킬작, 오버로드, 큐브, 소장품 메모가 “검수 필요”가 아님
- 출처 URL이 있고 원문 복사가 아닌 직접 요약 메모만 저장됨

## communityChecked로 올리는 기준

- 공식/인게임 기본 정보는 확인됨
- 최신 메타가 완전 확정은 아니지만 커뮤니티 평가가 대체로 안정적임
- 점수와 역할은 사용할 수 있으나 패치/보스 기믹에 따라 변동 가능성이 있음

## needsReview 유지 기준

- 신규, 한정, 콜라보, Treasure, 평가가 갈리는 캐릭터
- 스킬작/오버로드/큐브 추천이 아직 확실하지 않은 캐릭터
- 콘텐츠별 점수 일부가 임시값이거나 최신 메타와 차이가 날 수 있는 캐릭터

## placeholder 유지 기준

- 이름만 등록했거나 한국어명/기본 분류가 미확인인 항목
- `burst: "none"`, `weapon: "미확인"`, `element: "미확인"`처럼 전투 정보가 비어 있는 항목
- Nikke Deck 209명 카운트 정합성을 위해 먼저 만든 Treasure placeholder

## contentScores 점수 기준

- 90~100: 매우 강력하거나 핵심급
- 75~89: 자주 쓰이는 좋은 선택지
- 60~74: 조건부 사용 가능
- 40~59: 특정 상황 또는 대체용
- 0~39: 미검수, 부적합, 또는 해당 콘텐츠와 잘 맞지 않음

placeholder는 평균 점수 0으로 유지하고, 검수 후에만 점수를 올립니다.

## roles 태그 기준

`types/nikke.ts`의 `allowedRoles`에 있는 표준 태그만 사용합니다. 새 태그가 꼭 필요하면 타입과 검증 규칙을 함께 수정합니다.

## aliases 작성 기준

- 한국어명, 영어명, 띄어쓰기 없는 별칭, 한국 커뮤니티 약칭을 우선합니다.
- 확실하지 않은 별칭은 넣지 않습니다.
- 검색 편의를 위해 한 캐릭터당 최소 2~3개 이상을 목표로 합니다.

## 외부 출처 저장 주의사항

- 출처 URL은 `sourceUrls`에 남깁니다.
- 외부 사이트의 설명문, 공략문, 리뷰 문장을 그대로 복사하지 않습니다.
- 수치와 기본 분류는 확인 근거로만 사용하고, notes는 앱용으로 새로 요약합니다.
- 최신 메타가 불확실하면 `needsReview` 또는 `placeholder`를 유지합니다.
