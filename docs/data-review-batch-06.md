# 데이터 검수 배치 06

Date: 2026-07-01

## 목표

- 콘텐츠별 점수 체계 보정
- 추천 알고리즘 점수 균형 개선
- 과대평가/저평가가 의심되는 contentScores 패턴 진단

## 추가 파일

- `data/contentScorePolicy.ts`
- `lib/contentScoreAudit.ts`

## 점수 정책 요약

- story: 광역 처리, 버스트 순환, 유지력, 자동전투 안정성을 중점으로 봅니다.
- boss: 단일 보스 딜, 버프/디버프, 파츠 처리, 쿨감 기여를 중점으로 봅니다.
- tower: 기업 제한 환경에서도 쓸 수 있는 범용성, 버스트 순환, 생존 보조를 봅니다.
- arena: 버스트 충전, 선버스트, 도발/무적/피해분배, 광역 압박을 봅니다.
- soloRaid: 보스 딜, 버프/디버프, 쿨감, 유지력, 여러 덱 분산 가능성을 봅니다.
- unionRaid: 보스 딜, 디버프, 대체 딜러 활용성, 장기 운용 안정성을 봅니다.
- interception: 보스 딜, 파츠 처리, 저지/생존 보조, 버스트 안정성을 봅니다.

## 수정한 점수

이번 배치에서는 니케별 `contentScores` 숫자를 직접 대량 수정하지 않았습니다.

대신 다음 로직을 추가해 추천 결과에서 점수 숫자만 과하게 믿지 않도록 했습니다.

- 콘텐츠별 핵심 역할 정책
- 역할 적합도 보정
- sourceStatus 신뢰도 보정
- 콘텐츠별 팀 시너지 보정
- 의심 점수 패턴 진단

## 의심 패턴 목록

`lib/contentScoreAudit.ts`에서 아래 항목을 진단합니다.

- 모든 콘텐츠 85점 이상인 만능형 후보
- needsReview 상태인데 90점 이상 콘텐츠가 3개 이상인 후보
- 높은 콘텐츠 점수에 비해 핵심 역할 근거가 부족한 후보
- 아레나 역할 없이 arena 90점 이상인 후보
- 보스전 역할 없이 boss/soloRaid/unionRaid 90점 이상인 후보
- SSR인데 평균 점수가 30점 미만인 후보
- 평균 점수가 90점 이상인 후보

## 추천 알고리즘 영향

- `getNikkeCompositeScore`에 역할 적합도 점수를 추가했습니다.
- `getSourceStatusPenalty`에서 `communityChecked` 감점을 -2에서 -1로 완화했습니다.
- 팀 점수에 콘텐츠별 팀 시너지 점수를 추가했습니다.
- 추천 결과 카드에 기본 점수, 역할 점수, 역할 적합도, 팀 시너지, 신뢰도 보정을 접이식으로 표시합니다.

## 관리자 페이지 영향

`/admin/data-quality`에 콘텐츠 점수 진단 섹션을 추가했습니다.

표시 항목:

- 콘텐츠별 평균 점수
- 콘텐츠별 90점 이상 니케 수
- 콘텐츠별 70점 미만 니케 수
- 모든 콘텐츠 85점 이상 후보 수
- needsReview 고점 후보 수
- 의심 패턴 후보 수
- 콘텐츠/sourceStatus/90점 이상/의심 패턴 필터
